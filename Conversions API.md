# Conversions API

**Updated:** 25 de fev de 2026

The Conversions API is designed to create a connection between an advertiser’s marketing data, such as:

- Website events;
- App events;
- Business messaging events;
- Offline conversions.

This connection can be created from an advertiser’s:

- Server;
- Website platform;
- Mobile app;
- CRM.

The goal is to send this data to Meta systems that can:

- Optimize ad targeting;
- Decrease cost per result;
- Measure outcomes.

---

# How the Conversions API works

Rather than maintaining separate connection points for each data source, advertisers are able to leverage the Conversions API to send multiple event types and simplify their technology stack.

In the case of direct integrations, this entails establishing a connection between an advertiser’s server and Meta’s Conversions API endpoint.

---

# Server events

Server events are linked to a `dataset ID` and are processed like events sent using other Meta connection channels, such as:

- Meta Pixel;
- Facebook SDK for iOS;
- Facebook SDK for Android;
- Mobile measurement partner SDK;
- Offline event set;
- `.csv` upload.

This means that server events may be used in:

- Measurement;
- Reporting;
- Optimization.

They work in a similar way as events sent through other connection channels.

---

# Offline events

Offline events may be used for:

- Attributed offline events measurement;
- Offline custom audience creation;
- Measurement.

---

For optimal ad performance and measurement, Meta recommends that advertisers follow the Conversions API best practices.
